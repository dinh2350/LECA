Section 3 — Migration Phases

Bottom-up, 4 phases — each phase is shippable independently

PHASE 0 Infrastructure Setup ~1-2 days
+ Install Tailwind CSS + tailwind.config.ts
+ shadcn/ui CLI init
+ next-themes (dark/light toggle)
+ lucide-react (icons)
+ react-day-picker (date inputs)
+ @tiptap/react + extensions
+ cn() utility in src/lib/utils.ts
+ globals.css with AI dark design tokens
MUI stays untouched — this is purely additive
PHASE 1 UI Shell + Primitives ~2-3 days
→ Button, Badge, Avatar, Separator
→ Table, AlertDialog (ConfirmDialog)
→ AppBar → dark terminal-style header
→ Sidebar → custom Tailwind
→ SwitchThemeButton → next-themes
→ Snackbar → Sonner toast
→ MUI icons → lucide-react
→ FullPageLoader → Tailwind spinner
Layout looks different but pages still use MUI form components
PHASE 2 Form Components (27 components) ~1-2 weeks
→ TextInput → Input + Label [low]
→ Select / SelectExtended → Select [low]
→ MultipleSelect → Command + Popover [med]
→ Autocomplete → Combobox [med]
→ Checkbox / RadioGroup / Switch [low]
→ AvatarInput → custom upload [med]
→ ImagePicker → custom dropzone [med]
→ DatePicker / DateTime / Time [high]
→ Rich Text (mui-tiptap → @tiptap/react + custom toolbar) [high]
Each component migrated with Storybook story updated in same PR
PHASE 3 Pages + MUI Removal ~1 week
→ Auth pages (sign-in, sign-up, forgot-password)
→ Profile pages (view, edit)
→ Admin pages (users list, create, edit)
→ Confirm-email, privacy-policy pages
→ Remove MUI ThemeProvider + createTheme()
→ Remove Emotion registry.tsx
→ Uninstall @mui/* + @emotion/* packages
Final state: zero MUI dependencies
