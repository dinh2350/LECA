# Software Development Life Cycle (SDLC)

## SDLC Phases

### 1. Planning
- Define project scope, goals, timeline, budget
- Feasibility study (technical, financial, operational)
- Risk assessment and resource allocation
- **Deliverables:** Project Charter, Feasibility Report

---

### 2. Requirements Analysis
- Gather functional & non-functional requirements
- Stakeholder interviews, user stories, use cases
- Document in SRS (Software Requirements Specification)
- **Deliverables:** SRS, BRD (Business Requirements Doc)

---

### 3. System Design
- Architecture design (high-level & low-level)
- Database schema, API contracts, UI wireframes
- Tech stack selection
- **Deliverables:** Design Document, ERD, Architecture Diagrams

---

### 4. Implementation (Development)
- Code according to design specs
- Follow coding standards & code reviews
- Version control (Git branching strategy)
- **Deliverables:** Source Code, Unit Tests

---

### 5. Testing & QA
- Unit → Integration → System → UAT testing
- Bug tracking, regression testing
- Performance & security testing
- **Deliverables:** Test Plans, Bug Reports, QA Sign-off

---

### 6. Deployment
- CI/CD pipeline execution
- Staging → Production deployment
- Rollback plan in place
- **Deliverables:** Release Notes, Deployment Checklist

---

### 7. Maintenance & Operations
- Monitor performance, uptime, errors
- Patches, security updates, feature enhancements
- User feedback loop
- **Deliverables:** Incident Reports, Change Requests

---

## Popular SDLC Models

| Model | Best For |
|-------|----------|
| **Waterfall** | Fixed requirements, regulated industries |
| **Agile/Scrum** | Iterative delivery, changing requirements |
| **Kanban** | Continuous flow, ops/support teams |
| **DevOps** | Rapid deployment, automation-first |
| **Spiral** | High-risk, large enterprise projects |
| **V-Model** | Safety-critical systems (medical, aviation) |

---

## Industry Best Practices
- **Shift-left testing** — test early, not just at the end
- **CI/CD** — automate build, test, deploy pipelines
- **Definition of Done (DoD)** — clear completion criteria per phase
- **Traceability** — link requirements → code → tests
- **Security by design** — integrate security at every phase (DevSecOps)
